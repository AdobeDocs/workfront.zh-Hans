# 删除预览突出显示(Workfront)

## 范围

仅在&#x200B;**all**&#x200B;为true时应用：

1. 用户调用了此工作流（例如，显示&#x200B;**“删除预览突出显示”**&#x200B;或意图明显相同）。
2. Markdown文件的路径&#x200B;**不**&#x200B;包含&#x200B;**`product-announcements`**（排除整个文件夹树，如`help/quicksilver/product-announcements/`下的发行说明、betas和公告）。
3. Markdown文件是以下&#x200B;**排除的路径**&#x200B;下面列出的&#x200B;**[非](#excluded-paths)**。
4. Markdown文件的前置内容在&#x200B;**`Courtney`**&#x200B;行中包括`author:`（单一作者或共同作者）。
5. 文章具有&#x200B;**至少一个**：
   - 预览环境&#x200B;**正文语言或实际代码片段段落**（典型模式：“高亮显示的信息”、“预览环境”、“尚未公开发布”、快速发行说明） — **不**&#x200B;目录/索引页面上&#x200B;**仅链接文本**&#x200B;中的匹配项（见下文）；或
   - 任何具有&#x200B;**`class="preview"`**&#x200B;的HTML元素（例如`<span class="preview">`、`<div class="preview">`）；或
   - 预览片段&#x200B;**变量**，如&#x200B;**`{{highlighted-preview}}`**&#x200B;或&#x200B;**`{{highlighted-preview-article-level}}`**。

如果范围不明确，请先确认，然后再进行编辑。

**目录/索引页 — 始终跳过这种情况：** **从不**&#x200B;在清单上放置文件当&#x200B;**仅**&#x200B;预览相关的措辞&#x200B;**内部** markdown链接的显示文本指向&#x200B;**其他**&#x200B;文章（例如&#x200B;*在预览沙盒环境中发送报告*） **和**，文件有&#x200B;**no** `class="preview"`、**no**&#x200B;代码片段变量和&#x200B;**no**&#x200B;在&#x200B;**prose中的预览样板（链接外部）**。 这并非在该页面上预览突出显示，而只是目录提及。 适用于任何索引/目录，而不仅仅是一个文件。

### 排除的路径

切勿将它们添加到清单中或在此工作流中进行编辑，除非用户明确覆盖：

- `help/quicksilver/documents/adobe-workfront-for-experience-manager-assets-essentials/send-to-aem.md` — 并行预览与生产限制需要在自动化之外慎重作出编辑决定。
- `help/quicksilver/reports-and-dashboards/reports/creating-and-managing-reports/create-manage-reports.md` — 报告目录；唯一的“预览”信号是预览Sandbox交付文章的链接。

## 所需的工作流（循环中的人工）

未经批准，请&#x200B;**不**&#x200B;批量编辑存储库。

1. **库存**\
   构建符合上述范围规则的已排序路径列表（搜索存储库；首选`help/`个树）。 **忽略**&#x200B;下的任何路径&#x200B;**`product-announcements`**、**[排除的路径](#excluded-paths)**&#x200B;下的任何路径，以及范围下与&#x200B;**目录/索引页面**&#x200B;匹配的任何&#x200B;**目录/索引**&#x200B;页面。 如果用户说列出的文件没有预览突出显示，请从运行中删除它并收紧标准，而不是强制进行编辑。

2. **开始**\
   询问是否以列表中的&#x200B;**前**&#x200B;篇文章或用户名的路径开头。

3. **每篇文章 — 首先显示，然后编辑**
   - 读取文件。
   - 明确地提出编辑内容：**之前/之后**，或者针对将更改的内容提供重点突出的差异样式描述。
   - **等待**&#x200B;明确批准（例如，“确定”、“申请”、“是”），然后再写入文件。

4. **在每个文件之后**\
   询问是否移动到&#x200B;**下一个**&#x200B;文章（或停止/跳过）。

## 内容规则（GA：预览内容成为文档）

删除&#x200B;**常规可用性**&#x200B;的预览突出显示时，目标是：**保留为预览**&#x200B;记录的行为，删除&#x200B;**个过时的“生产中”/旧进程**&#x200B;分支，然后&#x200B;**删除仅预览标签和包装器**，以使所有客户的主题显示为当前主题。

### 并行步骤

- 如果文章有一个步骤（或编号项）**被框架设置为“生产中”**（或等效项：当前生产/现有生产行为）**和**，则有一个&#x200B;**并行**&#x200B;步骤，在预览中”**（或预览环境）被框架设置为**：
   - **删除**&#x200B;生产中的步骤（旧路径）。
   - **保留**&#x200B;预览中步骤的&#x200B;**substance**，但&#x200B;**重写**，因此它&#x200B;**不是**&#x200B;特定于预览（删除“在预览中”、“预览环境”等）。 调整编号，使列表保持一致。

如果结构模棱两可（没有清除的并行），则&#x200B;**停止**&#x200B;并显示两个候选项；询问用户保留哪个块。

### 平行截面

- 如果&#x200B;**部分** （标题+正文）是&#x200B;**关于“生产环境中”**，并且有&#x200B;**并行部分** **关于“在预览环境中”**：
   - **删除**&#x200B;生产环境部分（被取代的内容）。
   - **将**&#x200B;替换为预览部分的内容，然后&#x200B;**删除**&#x200B;预览环境措辞和任何&#x200B;**`class="preview"`**&#x200B;包装符，以使该部分为中性（GA就绪）。

### 代码片段和文章顶部声明

- 在功能正式发布后，移除&#x200B;**仅预览样板**&#x200B;块（跨越/div或仅说明高亮显示的内容为仅预览、快速发布、沙盒等的段落）。
- 删除其&#x200B;**仅**&#x200B;目的为预览可用性的链接或句子，除非用户要求保留其他通知。

### HTML `class="preview"`

- 对于使用&#x200B;**的元素，**&#x200B;保留内部内容&#x200B;**`class="preview"`**（标记内的markdown/HTML），请移除&#x200B;**开始和结束标记**。
- 处理&#x200B;**`<span class="preview">`**&#x200B;和&#x200B;**`<div class="preview">`**，以及其他标记（如&#x200B;**`<p class="preview">`**、**`<li class="preview">`**）上出现的相同模式(当它们出现在&#x200B;**可见** （非注释）正文内容中时)。
- 保留列表/表结构；修复断开的列表或展开后的零星空白。

### 注释掉的部分（HTML注释）

- **不要**&#x200B;删除、取消环绕或以其他方式&#x200B;**修改**&#x200B;**`<!-- … -->`** HTML中的任何内容备注&#x200B;**，除非用户明确指明了要执行的操作（例如，删除整个备注，仅取消备注中的预览类，取消对GA的备注）。**
- 如果预览相关内容或&#x200B;**`class="preview"`**&#x200B;仅&#x200B;**出现在评论中**，则&#x200B;**在查看文章时调用该内容**：说评论中的内容并&#x200B;**询问**&#x200B;要做什么。 **默认：保留备注节不变。**

### 不该做什么

- 请勿在&#x200B;**`product-announcements`**&#x200B;下的路径上运行此工作流（发行说明和相关内容）；清单必须排除它们。
- 请勿清点或编辑&#x200B;**[排除的路径](#excluded-paths)**&#x200B;下列出的路径，除非用户明确要求包含这些路径。
- **不**&#x200B;自动移除或编辑&#x200B;**注释掉的** (`<!-- … -->`)块；请遵循上述&#x200B;**注释掉的部分**。
- 当此功能可用性模式是&#x200B;**而不是**&#x200B;时，请勿删除“预览”（例如，在不相关的上下文中以[产品名称] (·)预览Sandbox环境&#x200B;**） — 使用判断并询问是否不确定。**
- 除非用户询问，否则不要更改`author:`或不相关的前置内容。
- 不要跳过&#x200B;**显示→批准**&#x200B;步骤。

## 呈现编辑内容之前的质量检查

- 议定的更改范围中没有剩余的&#x200B;**`class="preview"`**。
- 没有孤立的重复标题或损坏的步骤编号。
- 简介可正确阅读&#x200B;**而非**&#x200B;与GA冲突（对于已提供的功能，“仅在‘预览’中”）。
