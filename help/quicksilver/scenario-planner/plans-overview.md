## 计划概述

您可以确定每个高级别组织结果，并将其作为计划添加到 [!DNL Workfront Scenario Planner]. 计划是 [!DNL Scenario Planner]. 为了更便于实现计划，您可以将计划分为若干计划，以指明各个组织单位完成计划需要采取哪些步骤。

然后，您可以将计划与实际项目相连，以指示实际工作因素如何实际完成计划。 本文提供了有关计划的一般信息。 有关计划的信息，请参阅 [中的计划概述 [!DNL Scenario Planner]](../scenario-planner/initiatives-overview.md).

您可以定义完成计划上确定的工作所需的人工和财政资源。 您还可以查看计划中的计划所需的人工和财政资源，以完成这些计划。

您可以估计并复查每个计划的以下信息：

* 估计可用于执行计划的作业角色的类型和数量。
* 估算贵公司完成计划所需的预算。
* 根据与计划关联的所需工作角色，查看每个工作角色的使用百分比。
* 根据与计划相关的成本，查看计划预算的使用百分比。
* 在任何给定时间点查看计划的净值。
* 当您选择不同的方案时，计划层信息会发生更改。 每个方案具有不同的预算和人员信息。

有关创建计划的信息，请参阅文章 [在 [!DNL Scenario Planner]](../scenario-planner/create-and-edit-plans.md).

有关创建方案的信息，请参阅 [在 [!DNL Scenario Planner]](../scenario-planner/create-and-compare-scenarios-for-a-plan.md).

## 有关计划的注意事项

创建计划时请考虑以下事项：

* 您可以为团队、整个部门，甚至整个公司构建计划。 计划是大型规划单位，它概括了一个公司在高层的战略。
* 最短计划的持续时间为1年。 最长的计划可以有5年的期限。
* 不能对计划进行实际工作。 您可以粗略估计是否拥有开始规划工作所需的资源和预算。 例如，如果贵公司希望在新位置扩展并收购一个新办公室，那么完成该任务所需的步骤首先可以在高层管理层的计划中列出。
* 您可以创建同一计划的多个方案。 当原始计划存在太多冲突的计划时，您将需要创建多种方案，以编辑计划或预算及成本，以了解完成计划的理想情况。 当计划在同一时间范围内尝试使用相同的资源时，它们可能会相互冲突。 然后，您可以比较各种情景，以了解哪些情景最有意义以及贵公司应采用哪种情景，然后再开始添加实际工作以完成它。 有关创建方案的信息，请参阅文章 [在 [!DNL Scenario Planner]](../scenario-planner/create-and-compare-scenarios-for-a-plan.md).
* 您可以向计划中添加多个方案，以指明较小的规划单元将如何促进计划的完成。 例如，当您计划扩展到特定市场时，您可以在部门级别采取多项举措，最终有助于实现组织所有级别的扩展。 有关创建计划的信息，请参阅 [在 [!DNL Scenario Planner]](../scenario-planner/create-and-edit-initiatives.md).
* 创建计划时，您是唯一可以查看计划的人员。 您必须与其他用户一起制定计划，并且如果他们至少拥有 [!DNL Scenario Planner] 访问级别。

## 计划的职务职责信息

您可以复查计划的职务职责一般信息，以指明计划可用的职务职责数量以及计划完成所需的职务职责数量。 您可以在 [!UICONTROL 作业角色] 框。

![](assets/job-role-box-on-plan-not-expanded-fte-350x141.png)

您可以在作业角色框中查看以下内容：

| 信息 | 描述 |
|---|---|
| FTE/小时指示器 | ([!UICONTROL FTE])或([!UICONTROL 小时])指示器[!UICONTROL 作业角色]“ title指示计划是设置为使用FTE还是创建时的小时数。 计划、所有情景和计划都使用此单位。 |
| [!UICONTROL 可用] | 当前方案中可用的职务角色FTE或小时数。 |
| 必填 | 当前方案中所有方案所需的工作角色FTE或小时数，以便能够完成这些任务。 |

有关访问计划和查看有关作业角色的详细信息的信息，请参阅 [在 [!DNL Scenario Planner]](../scenario-planner/create-and-edit-plans.md).
