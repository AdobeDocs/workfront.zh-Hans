# 22.3项目增强功能

本页介绍了在“预览”环境中22.3版本所做的所有项目增强功能。 这些增强功能在2022年7月11日这一周的生产环境中提供。 有关22.3版本中可用的所有更改的列表，请参阅 [22.3版本概述](../../../product-announcements/product-releases/22.3-release-activity/22-3-release-overview.md).

## 外部工作代表团

现在，当您计划在短时间内离开办公室时，您可以将分配给您的任务和问题临时委派给其他用户。 这可以确保你的缺席不会成为完成工作的障碍。

在此增强之前，您只能委派批准。

以下是我们在此更新中添加的一些功能：

* 系统或组管理员在“设置”的“任务和问题首选项”区域中设置，以在您的环境中启用委派。

* 在“主页”区域中为具有“审阅”或更高许可证的用户提供了用于委派其工作项目的“委派任务和问题”的新选项。

* 在“主页”(Home)和“任务”(Assignments)区域的任务和问题标题中指示项目已委派给其他人。

* 设置区域中的事件通知和用户配置文件中的电子邮件通知，用于控制有关委派工作的电子邮件通知


>[!NOTE]
>
>只有具有审阅或更高许可证的用户才能将其工作委派给其他人。 工作可委派给任何用户，而不论其访问级别如何。 委派的用户将获得与受分配者对委派项目的相同权限。 如果这些权限低于用户的访问级别配置，则可能会阻止委派的用户对委派给他们的任务和问题执行某些活动。


有关更多信息，请参阅 [委派任务和问题概述](/help/quicksilver/manage-work/delegate-work/delegate-work-overview.md).

## 将问题转换为任务时的新体验

为了使您对Workfront的使用与新的Workfront体验保持一致，我们重新设计了将问题转换为任务的界面。

此更新包括：

* 更新的用户界面，与其余的新Workfront体验相匹配。

* 有权从列表或报表将问题转换为任务。

* 在添加到新任务的目标项目的“任务设置”区域中定义的默认自定义表单。


有关更多信息，请参阅 [在Adobe Workfront中将问题转换为任务](/help/quicksilver/manage-work/issues/convert-issues/convert-issue-to-task.md).

## 将问题转换为没有模板的项目时的新体验

为了使您对Workfront的使用与新的Workfront体验保持一致，我们重新设计了界面，无需使用模板即可将问题转换为项目。

除了与其余新Workfront体验相匹配的更新用户界面之外，我们还增加了从列表或报表将问题转换为空白项目的功能。

有关更多信息，请参阅 [在Adobe Workfront中将问题转换为项目](/help/quicksilver/manage-work/issues/convert-issues/convert-issue-to-project.md).

## 更新流中的智能标记

在您创建新更新或回复现有更新或回复时，我们改进了更新流中的标记用户。 现在，当您为用户添加标签以在更新中包含这些用户时，除了他们的姓名和头像外，我们还会显示他们的“主要角色”和电子邮件。 这有助于区分具有相似或相同名称的多个用户。

有关更多信息，请参阅 [更新时标记其他人](/help/quicksilver/workfront-basics/updating-work-items-and-viewing-updates/tag-others-on-updates.md).

## 自定义字段中计算的新语法

为了为将来的增强功能做好准备，帮助您将计算添加到自定义表单中，我们已对添加到计算中的引用字段的语法进行了标准化。 使用这种新语法很容易，因为当您开始键入字段名称，然后选择该字段时，系统会为您输入该语法。

有关更多信息，请参阅 [文章将计算数据添加到自定义表单](/help/quicksilver/administration-and-setup/customize-workfront/create-manage-custom-forms/add-calculated-data-to-custom-form.md).

## 当审批流程中涉及两个具有共同角色的用户时，请保留准确的信息

为了确保批准工作的数据准确性，当多角色批准流程与项目关联时，我们更改了在项目上记录批准信息的方式。

某些审批流程需要获得两个不同角色的批准，而两个不同的审批者可能具有相同的其中一个角色。 现在，在进行审批决策后，Workfront会记录每个审批者以及与审批流程关联的各自角色。

在进行此更改之前，会为第二个用户记录两个批准，因为他们与第一个批准者共享了其中一个批准角色。 在这种情况下，第二个审批者会覆盖第一个审批者的信息。

有关Workfront中审批流程的更多信息，请参阅 [审批流程概述](/help/quicksilver/review-and-approve-work/manage-approvals/approval-process-in-workfront.md).

## 在对分配进行更改时，将不再删除分配小时数

>[!NOTE]
>
>此功能最初计划在22.2版本中发布。 它于2022年4月21日发布到生产版。


为了确保数据的准确性，我们进行了相应更改，以在对任务上的分配进行更改后，保留分配小时数并保持任务计划小时数不变。

已对具有简单持续时间类型的任务进行以下更改：

* 删除所有分配时，将保留计划小时数。

* 替换用户和角色时，将保留单个分配分配。

* 删除用户时，角色上的单个分配将保留。 （从版本中删除）。 现在，在删除所有分配后，计划时数将设置为0。)


有关“计划时数”的详细信息，请参阅 [计划时数概述](/help/quicksilver/manage-work/tasks/task-information/planned-hours.md).

## 展示板增强功能

向Adobe Workfront展示板添加了以下增强功能：

* 筛选选项 — 您现在可以在展示板上按到期日期或状态进行筛选。 过滤器还更新了可折叠的部分，以分隔各个选项。

* 存档展示板 — 现在，您可以在展示板中存档展示板，而不必转到展示板仪表板。

* 将团队添加到展示板 — 在搜索成员时，可以添加整个团队。 选择团队会将团队中的每个人加入董事会。

* 在列中拖放区域 — 将信息卡从一个展示板列拖放到另一个展示板列时，灰色的“拖放区域”现在会显示信息卡的放置位置。 以前，卡片放置时没有可视指示器。

* 连接的卡 — 您现在可以添加与Workfront任务和问题连接的卡。 在卡片或任务中更新名称、描述或被分派人将更新其他位置中的相同字段。

* 所有卡片上的状态字段 — 在临时卡片和连接的卡片上都添加了状态字段和标记完成按钮。 单击“标记完成”后，状态将自动更改为“完成”。

* 将Ad Hoc卡转换为连接的卡 — 现在，您可以从卡详细信息中选择将Ad Hoc卡转换为连接的卡。

* 断开连接的卡 — 断开连接的卡会断开与Workfront对象的连接。 该卡在展示板上仍作为临时卡，并且Workfront对象不受影响。

* 列中的状态映射 — 新列设置和策略允许您定义应用于移动到该列的卡的状态、代理人或标记。 此外，新展示板上的默认列名称已更改为列1、列2和列3。

* 字段更新指示器 — 现在，在保存信息卡时会显示一个指示器，以确认已保存更新。


有关更多信息，请参阅 [开始使用Adobe Workfront中的展示板](/help/quicksilver/agile/get-started-with-boards/get-started-with-boards.md).

## 仅在文件夹层次结构的前五个级别中共享文件夹

为了确保用户共享文件夹的最佳性能，我们当前将共享限制为对象上文件夹层次结构中排名前五的级别。

第六级或以下级别的每个文件夹，都会从其正上方的文件夹继承其共享配置。

有关共享文件夹的更多信息，请参阅 [共享顶级文档文件夹](/help/quicksilver/workfront-basics/grant-and-request-access-to-objects/share-a-document-folder.md).

## Workfront Campaigns（测试版） — 一种管理工作的新方法

>[!NOTE]
>
>此功能计划于2023年1月9日从“预览”中删除。 有关更多信息，请参阅 [23.1版本概述页面](/help/quicksilver/product-announcements/product-releases/23.1-release-activity/23-1-release-overview.md).

>[!NOTE]
>
>22.3生产版本中将不包含此功能。 该版本将随后发布到生产环境。


>[!NOTE]
>
>此功能仅作为测试版提供，目前正在构建中。 我们将在未来版本中继续为Campaign工作流添加功能。 参与Workfront Campaigns测试版计划是自愿的。

我们将向Adobe Workfront引入一个新对象，该对象可能会更改您管理工作的方式。

Workfront促销活动允许您在新的工作容器中组织来自不同项目组合和项目的项目。 此新容器将在未来版本中发展，最终包含当前在单独的数据仓库中管理的所有工作对象。

此版本包含以下功能：

* 名为Campaign的新Workfront对象

* 主菜单中新的促销活动（测试版）区域

* “营销活动”区域的营销活动列表

* 营销活动详细信息页面，显示有关营销活动的其他信息

* 能够向营销活动添加项目

* 能够编辑有关营销活动的信息

* 能够从布局模板重命名Campaign对象

   Workfront系统和组管理员可以在布局模板的主菜单中添加营销活动（测试版）区域。 这样，该模板便可供分配给模板的所有用户使用。 在可用后，Workfront中的任何人都可以创建营销活动。
