# 写入质量引用

有关`writing-quality`技能的详细规则。 SKILL.md文件涵盖了最常见的规则；此文件提供了标点、术语、大写示例和过程格式的完整参考。

&#x200B;---

## 标点

### 期间

- 如果项目完成一个句子，则在列表中使用句点（“从X菜单中，您可以：”）
- 如果有任何项目是完整的句子，则在列表中使用句点
- 不得使用主内容设置句点的格式(正确：单击&#x200B;**完成**。 /不正确：单击&#x200B;**完成。**)
- 在文件扩展名(.pdf)中使用句点

### 逗号

- 牛津逗号始终：“动机、手段和机会”
- 介绍性短语后的逗号：“要打开文档，请单击“打开”。
- 年份后的逗号日期中句：“2016年1月2日，第……”期
- 过程中的“then”前面的逗号：“单击&#x200B;**设置**，然后单击&#x200B;**接口**。”
- 无仅包含月年的逗号：“2016年1月”

### 撇号

- 用于收缩：不要，他们是，他们是
- 单一占有性：添加“s”（Jen的计算机）
- 以s为结尾的复数占有：仅添加“（The Joneses”计算机）
- 区分“它”（它是）和“它”（属于它）

### 冒号

- 冒号后的小写，除非后跟直接引号、专有名词或项目符号列表
- 在引入列表的句子/片段末尾使用
- 过程标题后无冒号
- 无冒号可引入可视化元素或代码段

### 分号

- 连接两个子句而不使用连接
- 包含其标点符号的单独短语
- 在过程中不要在加粗文本后面加粗分号
- 不包含在超链接中

### 引号

- 将它们称为“引号”（而不是“引号”）
- 右引号位于逗号和句点之外
- 问号的位置取决于是引号还是句子是问题

### 连字符

- 尽可能避免使用需要连字符的结构
- 名词前加连字符复合修饰符：“只读内存”
- 当一个修饰符为分词时用连字符：“copy-protected disk”
- 当一个修饰符为数字时断字：“12点字体”
- 组合键中无连字符 — 使用+ (Ctrl+I)

### 长划线

请谨慎使用。 请考虑改组此句子。

### 短划线

- 范围：2005-2013年
- 负号或负数
- 将复合形容词连接到另一个单词：“dialog box-type options”

### 椭圆

- 省略了代码或语法
- UI指示将显示对话框
- 引号中省略了文本（替换句末时添加句点）

### Slash

- 没有用于选择的斜杠 — 使用“或”（“他/她”而不是“他/她”）
- 用于组合：打开/关闭开关
- 用于Internet地址和部分

## 大写示例

### Workfront术语 — 直接引用与间接引用

| 术语 | 直接（大写） | 间接（小写） |
|------|---------------------|----------------------|
| 规划完成日期 | “完成&#x200B;**计划完成日期**&#x200B;字段。” | “每项任务都必须有计划的完成日期。” |
| 布局模板 | “在‘设置’中，转到&#x200B;**布局模板**。” | “为每个用户分配一个布局模板。” |
| 组管理员 | “找到&#x200B;**组管理员**&#x200B;字段。” | “请联系您的组管理员。” |
| 自定义表单 | “将&#x200B;**自定义表单**&#x200B;附加到项目。” | “您可以为任何对象创建自定义表单。” |

### 常用Workfront术语（引用UI时大写）

- 自定义表单
- 持续时间
- 规划小时数
- 规划完成日期
- 小时数类型
- 组管理员/系统管理员
- Workfront管理员（人员，始终使用小写“管理员”）
- 提醒通知
- 布局模板
- 筛选器、视图、分组（在筛选器、视图和分组的上下文中）
- 资源规划者（计划）

### 按钮

- 遵循用户界面大小写
- 如果UI中的按钮全部大写，请改用句子大小写形式写

## 术语

### UI元素

| 元素 | 正确的术语 |
|---------|-------------|
| 可折叠的选项列表 | 菜单（添加“下拉列表”以便在需要时更清晰明了） |
| 命令不可用 | “不可用”（从不“变暗”、“已禁用”或“灰色”） |
| 包含表单字段的弹出窗口 | 对话框 |
| 未命名的按钮 | 使用按钮的工具提示名称 |
| 错误消息/Toast通知 | “消息”（“显示一条消息通知您……”） |
| 文本输入 | 字段 |

### 角色

| 上下文 | 术语 |
|---------|------|
| Workfront中的角色 | 系统管理员 |
| Workfront的管理者 | Workfront管理员 |

### 覆盖与覆盖

这些是含义不同的不同单词 — 请使用正确的单词来表示上下文。

### 文件扩展名

始终使用文件扩展名，而不是首字母缩略词：“上传.pdf文件”（而不是“上传PDF”）。

## 过程格式

### 介绍

仅在必要时包含。 首选格式（按顺序）：

1. 不定式短语+冒号：“要为系统中的用户创建临时密码：”
2. 完整的句子+句点：“您必须为系统中的用户创建临时密码。”
3. 不定式短语+“按照以下步骤操作”+冒号：“要为用户创建临时密码，按照以下步骤操作：”

### 步骤约定

- 对所有过程使用编号列表，即使是单步过程
- 步骤中的粗体可点击操作和UI元素
- 使用特定操作动词：“type”、“select”、“click” — 避免“set”或“specify”
- 描述系统响应在发生之前或发生之后立即发生的响应

系统响应放置示例：

```
1. Click **Sharing**.
2. In the **Task Access** dialog box, enter the name of the user you want to give task access to.
```

### 有条件和可选步骤

```
3. (Conditional) If you are a member of more than one team, select...
3. (Optional) To add an emoji, click...
3. (Conditional and optional) If you want to add a note...
```

### 子步骤和字段列表

使用命令导入（配置、指定、执行），然后列出字段：

```
1. Specify the following:
   a. **Favorite color**: Select your favorite color from the drop-down menu.
   b. **Favorite song**: Type your favorite song.
```

### 多种方法

首先记录最直接的方法：

```
1. In the **Main Menu**, click **Home**.

   Or

   Click the **Home** icon in the upper-left corner.
```

### 右尖括号

- 显示菜单/选项卡导航：“单击&#x200B;**电子邮件** > **通知**”
- 粗体按钮名称，而不是括号
- 括号中的空格
- 请勿用于主菜单 — 请使用主菜单片段

## 粗体和斜体参考

### 粗体 — 用于：

| 上下文 | 步骤 | 外部步骤 |
|---------|----------|---------------|
| 可单击的按钮/链接 | 粗体 | 不要粗体 |
| 菜单名称 | 粗体 | 不要粗体 |
| 图标名称 | 粗体 | 不要粗体 |
| 对话框名称 | 粗体 | 不要粗体 |
| 页面名称 | 粗体 | 不要粗体 |
| 字段列表中的字段名称 | 粗体+冒号 | 不要粗体 |

### 斜体 — 用于：

- 文本用户应键入：“在URL框中键入&#x200B;*my.workfront.com*”
- 从不强调 — 改改编句子

## 注释、提示、警告 — 详细指南

### 何时使用每个

**注释** (`>[!NOTE]`)：
- 不符合段落上下文的信息
- 避免用户受挫或浪费时间（“您必须拥有团队许可证才能完成此任务。”）
- 版本控制或兼容性信息（“此功能仅在旧版验证查看器中可用。”）

**提示** (`>[!TIP]`)：
- 使用户生活更轻松的建议并不重要
- 示例：“您可以通过按住Shift键同时共享多个文档。”

**重要** (`>[!IMPORTANT]`)：
- 步骤或过程的重要信息

**警告** (`>[!WARNING]`)：
- 提醒用户可能发生的数据丢失

### 规则

- 每部分最多一个
- 从不栈叠多个注释/提示/警告
- 将相关备注合并为带项目符号列表的备注
- 不要使用注释来声明新功能 — 重写文章文本

## 交叉引用阵列

| 状况 | 模式 |
|-----------|---------|
| 提供信息后 | “有关详细信息，请参阅[文章名称] (link)。” |
| 无事先信息 | “有关信息，请参阅[文章名称] (link)。” |
| 到另一篇文章中的章节 | “有关详细信息，请参阅[文章名称] (link)中的[节]。” |
| 到同一文章中的章节 | “有关详细信息，请参阅本文中的[节]。” |

投放位置：
- 在段落中：内联
- 在过程步骤中：步骤后面的新行
- 多个引用：项目符号列表

## DQTI质量特性

### 易于使用

| 特征 | 关键问题 |
|---------------|---------------|
| 任务方向 | 这是否侧重于用户可执行的操作？ 标题是否是命令动词？ |
| 准确度 | 术语是否与UI完全匹配？ 技术细节是否正确？ |
| 完整性 | 是否有足够的信息来采取行动？ 是否存在要删除的不必要信息？ |

### 易于理解

| 特征 | 关键问题 |
|---------------|---------------|
| 清晰度 | 每一句句子都是一个主意吗？ 文字简单直接吗？ |
| 具体 | 有具体的例子吗？ UI元素的命名准确吗？ |
| 样式 | 主动语态？ 现在时态？ 术语是否一致？ |

### 易于查找

| 特征 | 关键问题 |
|---------------|---------------|
| 组织 | 逻辑结构？ 渐进式披露？ 适用时最新优先？ |
| 可检索性 | 读者能否扫描标题并理解内容？ 是否提供链接？ |
| 视觉效果 | 3个以上项目的列表？ 要比较的表格？ 没有文本墙段落？ |

## 常见修复

| 问题 | 修复 |
|-------|-----|
| “我们很高兴能介绍……” | 删除营销语言 — 直接说明功能 |
| “更新了X的配置” | “您现在可以配置X”（主动、面向任务） |
| “建议……” | “我们建议……” |
| “用户可以利用这项功能……” | “您可以使用此功能……” |
| &quot;set the value&quot; / &quot;specify the option&quot; | &quot;键入值&quot; / &quot;选择选项&quot; |
| &quot;disabled&quot; / &quot;grayed out&quot; | &quot;unavailable&quot; |
| &quot;dropdown&quot;（独立） | “菜单”或“下拉菜单” |
| “他/她” | “他们”或重组 |
| 栈叠笔记 | 用项目符号合并为一个注释 |
| 粗体外部过程步骤 | 删除粗体 |
